Приложение для управления контактами участников на PHP 8.3 + MySQL 5 + Laravel Filament 12

Приложение должно работать на мобильных телефонах и на десктопе в браузерах и иметь поддержку двух языков: русский (основной) и английский

У пользователя системы есть 4 роли: лидер, менеджер, администратор, суперадмин. Пользователь одновременно может иметь одну, несколько или даже все роли.

Каждая роль имеет доступ к своей вкладке / странице со своими операциями + есть отдельная вкладка с дашбордом с общей статистикой и 10 (настраивается суперадмином в настройках системы) последними добавленными контактами. доступ к дашборду может быть предоставлен администратором любой роли

На главной странице приложения форма авторизации по почте (с валидацией при вводе) и паролю. Если пользователь не зарегистрирован ещё в системе, то есть ссылка на страницу регистрации, где он должен ввести свою почту (валидируется при вводе) и придумать пароль. После заполнения формы и отправки на регистрацию в системе пользователю должно показываться окно с информацией о том, что его контакт требует подтверждения администратором системы, а емейл подтверждения через почту. Если почта пользователя подтверждена и он авторизован администратором пользователя попадает в систему. По умолчанию любому пользователю присваивается роль лидера.


Требуется изготовать первоначальное заполнение БД с помощью сидинга и внести первого пользователя, имеющего все роли и доступ к Дашборду, в систему с уже подвердённой почтой и авторизованного. С него начнётся функционирование системы.

Все действия пользователей системы должны логироваться в таблицу БД по аудиту системы с отметками времени и привязке к пользователю

Функции ролей пользователя:

Менеджер: может создавать контакты, изменять данные контактов в системе, удалять контакты. Менеджер может самостоятельно назначать ответственного лидера за обработку данного контакта (жлательно иметь контекстное меню с операциями над контактом). Контакт включает следующие поля: ФИО (обязательное поле), телефон (обязательное поле с валидацией что это телефон), е-мейл (с валидацией что это емейл), округ, комментарии с отметкой времени, история изменения статусов с временной меткой и id пользователя выполнившего это действие. Менеджер может искать контакт по номеру телефона, по ФИО, по округу. В случае назначения контакта менеджером лидеру, лидеру на почту уходит уведомление об этом.

Лидер: не может редактировать контакт, но может изменять статус контакта и добавлять комментарии. Ему доступен список контактов с возможностью фильтровать контакты по различным параметрам. При входе в систему лидер попадает во вкладку/страницу со списком контактов, где он назначен ответственным (галочка мои контакты применена) и которые ещё не имеют статуса контакт обработан ( в обоих вариантах - успех/неуспех). Если снять галочку мои контакты, то лидер должен видеть все контакты.

Администратор: выдаёт разрешения на доступ в систему, выдаёт разрешения на доступ к дашборду, создает пользователя, подтверждает пользователя, блокирует пользователя, удаляет пользователя, назначает роли пользователям. При первоначальном создании пользователя администратором галочка роли лидер должна быть включена по умолчанию.

Суперадмин: задаёт общие параметры системы: 
- таймаут просрочки обработки контакта в днях (по умолчанию 30 дней)
- настройки почтовые сервера рассылки емейл уведомлений
- число записей в списке новых пользователей, добавленныъ в систему для отображения на дашборде (по умлочанию 10)

Жизненный цикл контакта: 
1. контакт не обработан
2. назначен исполнитель
2.bis исполнитель просрочил обработку контакта
3.А контакт обработан - результат успешен (конец цикла)
3.B контакт обработан - результат неуспешно (конец цикла) - может быть переведён менеджером в статус 1 или 2


Разработка будет вестись на локальной ПК с развернутой БД в Докере
