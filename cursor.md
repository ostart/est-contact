Приложение для управления контактами участников на PHP 8.3 + MySQL 5 + Laravel Filament (max последняя версия, совместимая с данной конфигурацией PHP и MySQL)

- Приложение должно работать на мобильных телефонах и на десктопе в браузерах и иметь поддержку двух языков: русский (основной) и английский

- У пользователя системы есть 4 роли: лидер, менеджер, администратор, суперадмин. Пользователь одновременно может иметь одну, несколько или все роли.

- каждой роли должна соответствовать своя страница в Sidebar слева: Лидер - Контакты, Менеджер - Управление, Администратор - Пользователи, Суперадми - Настройки

- Каждая роль имеет доступ к строго к своей странице со своими операциями + есть отдельная вкладка с дашбордом. Доступ к дашборду может быть предоставлен администратором любой роли.

- На дашборде размещается с общая статистика работы с контактами (сколько зарегистрировано, сколько обработано, сколько в текущей работе, сколько внесено новых контактов за последнюю неделю и т.п.). 

- На главной странице приложения располагается форма входа в систему по почте (с валидацией при вводе) и паролю. Если пользователь не зарегистрирован ещё в системе, то есть ссылка на страницу регистрации, где он должен ввести свою почту (валидируется при вводе) и придумать пароль. После заполнения формы и отправки на регистрацию в системе пользователю должна показываться страница с информацией о том, что его контакт требует подтверждения администратором системы, а емейл подтверждения отправлен на указанную почту. Если почта пользователя подтверждена и он авторизован администратором, то пользователь входит в систему. По умолчанию любому пользователю при создании / регистрации присваивается роль лидера.

- Требуется создать первоначальное заполнение БД с помощью сидинга и внести первого пользователя с почтой admin@example.com. Этот пользователь имеет все роли и доступ к Дашборду и уже подвердённую почту и разрешение администратора в БД. С него начнётся функционирование системы.

- Все действия пользователей системы должны логироваться в таблицу БД по аудиту системы с отметками времени и привязке к пользователю и выполненной им операции

Функции ролей пользователя:

Менеджер: может создавать контакты, изменять данные контактов, удалять контакты. Менеджер может самостоятельно назначать ответственного лидера за обработку данного контакта. Контакт включает следующие поля: ФИО (обязательное поле), телефон (обязательное поле с валидацией что это телефон), е-мейл (с валидацией что это емейл), округ, комментарии с отметкой времени, историю изменения статусов с временной меткой и id пользователя выполнившего это действие. Менеджер может искать контакт по номеру телефона, по ФИО, по округу. В случае назначения контакта менеджером лидеру, лидеру на почту уходит уведомление об этом.

Лидер: не может редактировать контакт, но может изменять статус контакта (такм образом беря контакт в работу) и добавлять комментарии. Удалять комментарии он не может. Ему доступен список контактов с возможностью фильтрации контактов по различным параметрам (ФИО, телефон, округ, статус, испольнитель). При входе в систему лидер попадает на страницу со списком контактов, где он назначен ответственным (галочка мои контакты применена) и которые ещё не имеют статуса контакт обработан (в обоих вариантах - успех/неуспех). Если снять галочку мои контакты, то лидер должен видеть все контакты и иметь возможность фильтрации по ним.

Администратор: выдаёт разрешения на доступ в систему, выдаёт разрешения на доступ к дашборду, создает пользователя, разрешает доступ пользователя в систему, блокирует пользователя, удаляет пользователя, назначает роли пользователям. При первоначальном создании пользователя администратором галочка роли лидер должна быть включена по умолчанию.

Суперадмин: задаёт общие параметры системы: 
- таймаут просрочки обработки контакта в днях (по умолчанию 30 дней)
- настройки почтовые сервера рассылки емейл уведомлений

Жизненный цикл контакта: 
1. контакт не обработан
2. назначен исполнитель
2*. исполнитель просрочил обработку контакта
3.А контакт обработан - результат успешен (конец цикла)
3.B контакт обработан - результат неуспешно (конец цикла) - может быть переведён менеджером в статус 1 или 2
